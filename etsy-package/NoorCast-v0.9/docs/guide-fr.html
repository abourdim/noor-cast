1 Démarrage Rapide

1.1 Qu'est-ce que NoorCast ?

NoorCast est un enregistreur d'écran multi-caméra gratuit, basé sur le navigateur spécialement conçu pour les enseignants créant des tutoriels vidéo avec des robots et des micro:bits. Il combine la capture d'écran, les flux webcam, l'entrée microphone et les données de capteurs micro:bit en un seul enregistrement — le tout depuis votre navigateur, sans rien installer.

Contrairement aux enregistreurs d'écran traditionnels, NoorCast a été conçu pour l'éducation. Il comprend qu'un enseignant expliquant la robotique a besoin de montrer son écran (code), son robot (caméra), son visage (webcam) et ses relevés de capteurs micro:bit — le tout simultanément, en changeant de disposition de manière fluide.

Points Clés

  • 100% local — rien ne quitte votre ordinateur. Pas de cloud, pas de compte, pas de tracking.
  • Zéro installation — ouvrez simplement une URL dans Chrome ou Edge. Fonctionne hors-ligne après le premier chargement.
  • Zéro dépendance — pas de npm, pas de build. Un fichier HTML + un JS + un CSS.
  • 200+ fonctionnalités — from basic recording to AI co-hosts and instant replay.
  • Trilingue — Français (par défaut), Anglais, Arabe (avec support RTL complet).
  • PWA installable — ajoutez à votre bureau/écran d'accueil comme une application native.

1.4 Mode Simple vs Pro

🎓 Simple Mode

Conçu pour les débutants et les jeunes élèves :

  • Barre latérale droite (scènes) repliée automatiquement
  • More Tools menu déroulant masqué
  • Boutons Pause/Marqueur masqués
  • Advanced settings sections masquées

🚀 Pro Mode

Accès complet à toutes les fonctionnalités :

  • Right sidebar with scenes visible
  • Tous les toolbar boutons accessibles
  • Full source style popup
  • Tous les settings sections visibles

Le mode est sauvegardé dans le localStorage. NoorCast se souvient de votre choix entre les sessions.

1.5 Votre Premier Enregistrement

Ajoutez une source écran. In the left sidebar, click the blue 🖥 + Add button. Chrome demande quel écran/fenêtre/onglet partager. Voir Chapitre 4.
Ajoutez une caméra. Sélectionnez votre webcam dans le menu déroulant, cliquez sur le bouton vert 🎥 + Add. Pour utiliser un téléphone comme caméra, voir Chapitre 3.
Sélectionnez un microphone. Choisissez dans le menu "Micro". A mic check s'exécute automatiquement.
Choisissez une scène. Cliquez sur un scene preset in the right sidebar: 💻 Code, 🤖 Mecha, etc.
Appuyez sur Enregistrer. Cliquez sur 🔴 REC ou appuyez sur R. A 3-2-1 countdown se lance, puis l'enregistrement commence.
Présentez. Changez de scène avec 1-7. Use overlays: L laser, F gel, D dessin. Appuyez sur M pour les marqueurs.
Arrêtez. Cliquez sur ⏹ Stop ou appuyez sur R. The take panel s'ouvre.
Téléchargez. Cliquez sur ⬇️ Télécharger. Ou utilisez les post-recording tools: Replay Instantané, Miniature, Transcription, Score Qualité.

3 Configuration Caméra — Guide Détaillé

NoorCast supporte plusieurs caméras simultanées. Ce chapitre couvre chaque façon d'ajouter une caméra, des webcams intégrées à l'utilisation de votre téléphone.

3.1 Webcam Intégrée du Portable

In the left sidebar, find the Caméra dropdown.
Votre webcam intégrée apparaît sous le nom "FaceTime HD Caméra" (Mac), "Integrated Caméra" (Windows), or similar.
Sélectionnez-la dans le menu déroulant.
Cliquez sur le bouton vert 🎥 + Ajouter.
Accordez la permission caméra quand le navigateur le demande.
Votre visage apparaît sur le canevas. Par défaut, il est placé comme un petit cercle en bas à droite (position PIP).
Si le flux caméra est inversé (le texte apparaît en miroir), cliquez-droit sur la source → Miroir pour corriger.

3.2 Webcams USB Externes

Plug in a USB webcam. It appears in the Caméra dropdown automatically (may take 2-3 seconds).

Using Multiple USB Caméras

NoorCast supporte un nombre illimité de caméras simultanées. Ajoutez-les une par une :

  1. Select Caméra 1 from dropdown → click Add
  2. Select Caméra 2 from dropdown → click Add
  3. Repeat for Caméra 3, 4, etc.

Each camera becomes a separate source on the canvas that you can position, resize, and style independently.

3.3 Téléphone comme Caméra (Webcam Sans Fil)

Utilisez votre iPhone ou téléphone Android comme caméra sans fil. Plusieurs options :

Option A: iPhone Continuity Caméra (Mac only)

Si vous avez un Mac sous macOS Ventura (13) ou ultérieur et un iPhone sous iOS 16+ :

Assurez-vous que les deux appareils sont connectés au même Apple ID et au même réseau Wi-Fi.
Le Bluetooth doit être activé sur les deux appareils.
In NoorCast, open the Caméra dropdown. Your iPhone appears as a camera option.
Sélectionnez-le et cliquez sur Ajouter. Le flux caméra de votre iPhone apparaît sur le canevas.
Placez votre iPhone près du robot pour une caméra robot dédiée. Utilisez la webcam intégrée de votre portable pour votre visage.

Option B : DroidCam (Windows/Mac + Android/iPhone)

Install DroidCam sur votre téléphone (gratuit sur App Store / Play Store).
Installez le client DroidCam sur votre ordinateur (Windows ou Mac).
Connectez votre téléphone et votre ordinateur au même réseau Wi-Fi.
Ouvrez DroidCam sur votre téléphone — il affiche une adresse IP et un port.
Ouvrez le client DroidCam sur votre ordinateur et entrez l'IP/port. Cliquez sur Démarrer.
In NoorCast, the Caméra dropdown now shows "DroidCam" as an option. Select and Add.

Option C : EpocCam (iPhone + Windows/Mac)

Install EpocCam sur votre iPhone.
Installez le pilote EpocCam sur votre ordinateur.
Les deux appareils doivent être sur le même Wi-Fi ou connectés par USB.
Ouvrez EpocCam sur votre iPhone.
In NoorCast, "EpocCam" appears in the Caméra dropdown. Select and Add.

Option D: OBS Virtual Caméra

Si vous utilisez déjà OBS Studio, vous pouvez :

In OBS, configurez le flux caméra de votre téléphone (via NDI, Spout, ou carte d'acquisition).
Cliquez "Start Virtual Caméra" in OBS.
In NoorCast, "OBS Virtual Caméra" appears in the dropdown. Select and Add.

3.4 Angles de Caméra pour les Tutoriels

Pour un tutoriel de robotique, vous avez généralement besoin de 2-3 angles de caméra :

📷 Face Caméra

Position : Eye level, looking at you.

Objectif : Shows your face while explaining. Builds connection with students.

Scène : Appears as a small circle (PIP) in most scenes. Full screen in "Solo" scene.

Conseils : Good lighting on your face. Neutral background. Look at the camera, not the screen.

🤖 Robot Caméra

Position : Overhead or angled view of the robot/breadboard.

Objectif : Shows the physical robot, wiring, or components.

Scène : Full screen in "Mecha" scene. Right half in "Code+Mecha".

Conseils : Use a phone on a tripod or clamp. Good lighting on the workspace. Avoid shadows from your hands.

🎛 Sensor Caméra

Position : Pointing at a display, oscilloscope, or sensor readout.

Objectif : Shows real-time sensor data or test equipment.

Scène : Full screen in "Sensors" scene.

Conseils : Use a second USB webcam or phone. Frame the display tightly.

🖥 Écran (pas une caméra)

Position : N/A — captures your computer screen.

Objectif : Shows code, IDE, browser, or presentation.

Scène : Full screen in "Code" scene. Left half in "Code+Mecha".

Conseils : Increase font size in your IDE. Fermez notification popups. Use a clean desktop.

4 Sources — Ajouter du Contenu

4.1 Capture d'Écran

Cliquez sur le bouton bleu 🖥 + Ajouter. Chrome propose trois options :

  • Écran Entier — capture tout sur votre moniteur
  • Fenêtre — capture une fenêtre d'application spécifique
  • Onglet Navigateur — capture un onglet Chrome spécifique (inclut l'audio de l'onglet)
Capturing a browser tab allows you to include the tab's audio (YouTube, online IDE) in your recording. The audio is automatically ducked when you speak. Voir Audio chapter.

4.2 Caméra

For detailed camera setup including phones, see Chapitre 3.

4.3 Microphone

Select from the "Mic" dropdown. Auto-activates immediately. Features:

  • Auto mic check — 3-second test: "🎤 Speak to test..." → "✅ Mic OK!" or "⚠️ Nothing detected". Voir Section 17.5.
  • Noise gate — mutes below threshold (-50 dB default). Configurable in Settings.
  • Gain boost — 0-4× slider in Settings.
  • Volume ducking — screen audio reduced when you speak. Voir Section 17.3.

  • Paste: Ctrl+V — paste any image from clipboard
  • Glissez & Drop: glissez an image file from desktop onto the canvas. Green dashed border highlights the drop zone. Supports PNG, JPG, GIF, SVG, WebP.

9.2 How Replay Instantané Works

Decodes audio from your recording using OfflineAudioContext
Computes RMS energy in 1-second windows across entire duration
Finds the top 7 loudest moments (non-overlapping, 4s clips)
Sorts clips chronologically
Re-encodes via canvas + MediaRecorder with flash transitions + clip counter badge
Downloads ~28s highlight reel as .webm + confetti burst

9.3 How Score Qualité Works

Analyzes 5 dimensions:

🎤 Audio (clarity)Average RMS level — higher = clearer voice
📈 Pacing (variation)RMS variance — varied = engaging, flat = monotone
🎭 Scènes (variety)Number of chapter markers = more scene switching
⏱ Duration (sweet spot)2-5 minutes scores highest. Too short/long penalized.
🔥 Engagement (peaks)% of windows above 2× average energy

Each dimension: A+ to D grade with colored progress bar. Overall = average. Score ≥80 triggers confetti.

29 Mode Reels — enregistrer pour les plateformes verticales (v0.9.x)

La plupart des utilisateurs de NoorCast filment en 16:9 paysage pour YouTube et les écrans d'ordi. Mais Facebook Reels, Instagram Reels, TikTok et YouTube Shorts sont tous en 9:16 vertical — uploader un clip paysage, c'est récolter d'énormes bandes noires (≈30% d'écran perdu) ou un recadrage automatique qui coupe vos sources. La série v0.9.x corrige ça de bout en bout.

29.1 Ce que le mode Reels apporte

Canvas portrait 9:16
Réglages → 📐 Format de la scène → 9:16 — 1080×1920. Le canvas, la prévisualisation et l'enregistrement basculent en vertical. Vos sources se réorganisent automatiquement.
📱 Guide safe-zone Reels
Nouveau bouton dans la barre d'outils, à côté de 📊 Sensors et 🎛 Gauges. Active un masque rouge semi-transparent sur les trois zones où l'UI de la plateforme couvre la vidéo : header haut (~13%), légende + boutons bas (~31%), colonne d'actions droite (~14%). La boîte verte en pointillés au milieu, c'est la safe-zone — gardez le contenu important dedans. Le guide reste visible uniquement dans l'éditeur, jamais dans l'enregistrement.
MP4 automatique en 9:16
FB et IG ré-encodent le WebM avec une perte visible. En 9:16, NoorCast préfère automatiquement H.264/MP4, même sur Windows.
Sous-titres dans la safe-zone
Les sous-titres live remontent à ~34% du bas en portrait pour rester lisibles après upload.
Alerte 90 s en direct
FB Reels, IG Reels, TikTok, YT Shorts plafonnent tous à 90 s. En mode 9:16, le chrono devient rouge et clignote au-delà, avec un toast à 90 s puis tous les 30 s.
📱 Couper à 90 s pour Reels (trim in-browser)
Si votre prise dépasse 90 s en 9:16, un bouton "Couper à 90 s pour Reels (−Xs)" apparaît dans le panneau de prise après l'arrêt. Un clic ré-encode les 90 premières secondes à 8 Mbps via le pipeline canvas + MediaRecorder utilisé par Silence-Trim et Instant-Replay. Aucune dépendance externe.
Helper ffmpeg
Pour un trim stream-copy (sans ré-encodage), un toast cliquable après chaque sauvegarde 9:16 copie une commande ffmpeg -ss 00:00:00 -to 00:01:30 -i <file> -c copy <file-reels> dans le presse-papier.
Saut d'intro audio
Réglages → 🎥 Capture → 🎙 Sauter le silence d'intro (démarrer au premier mot). Après le compte à rebours, NoorCast écoute votre micro et attend jusqu'à 5 s votre premier mot (RMS > 0,05) avant de lancer MediaRecorder. Plus de bla-bla mort en début de prise.

29.2 Workflow — première Reel, de A à Z

  1. Réglages → 📐 Format de la scène → 9:16 — 1080×1920. Le canvas bascule, un toast "📱 Mode Reels" apparaît, le bouton 📱 Reels s'allume.
  2. Regardez les zones rouges. Tout ce qui s'y trouve sera caché par l'UI FB/IG/TikTok. Glissez watermark, overlay capteurs, jauges servo dans la boîte verte.
  3. (Optionnel) Réglages → 🎥 Capture → 🎙 Sauter le silence d'intro.
  4. REC. Surveillez le chrono. S'il rougit, vous avez dépassé 90 s.
  5. STOP. Le panneau de prise s'ouvre. Si la prise dépasse 90 s, un bouton rose 📱 Couper à 90 s pour Reels (−Xs) apparaît. Un clic, un fichier -reels.mp4 se télécharge.
  6. Upload sur Facebook Reels. Choisissez le fichier -reels.mp4. Plus de bandes noires, plus de recadrage, sous-titres visibles, sous la limite des 90 s.

29.3 Conseils pour de bons Reels

  • Les 3 premières secondes comptent. Activez Sauter intro, commencez par l'action.
  • Sweet-spot 15–30 s. 90 s c'est le plafond, pas la cible.
  • Pré-visionnez en muet. ~70% des viewers Reels scrollent sans son — les sous-titres burned-in sont essentiels.
  • Composition verticale. Pensez au préset 👋 You (face-cam plein écran) ou à un layout perso (face en haut + contenu en bas).
  • Bitrate. Le défaut High — 8 Mbps est parfait pour Reels.

29.4 Limites connues

  • Le trim in-browser est un ré-encodage temps-réel (90 s de vidéo ≈ 90 s de trim). Pour un trim instantané, utilisez la commande ffmpeg.
  • Le bouton ne gère que les 90 premières secondes — pour un autre point de départ, éditez le -ss dans la commande ffmpeg.
