❓ NoorCast FAQ

Frequently asked questions about NoorCast

What is NoorCast?

NoorCast is a browser-based tool for recording multi-camera tutorial videos. Screen + webcams + mic, all from Chrome or Edge. Zero install, zero account, everything stays on your computer.

How do I add multiple cameras?

In the left sidebar, select a camera from the dropdown and click + Add. Repeat for each camera. There is no limit — you can add built-in webcams, USB cameras, and phones via DroidCam/Iriun/Camo. See the User Guide for phone setup instructions.

Can I use my phone as a camera?

Yes! Install a virtual webcam app on your phone:

Iriun Webcam (recommended) — free, all platforms, supports portrait mode
DroidCam — free, landscape only
Camo — pro quality, free/paid
Continuity Camera — Mac + iPhone, zero install

Full setup instructions: guide.html → Phone Cameras

Can I get a vertical/portrait camera feed?

Use Iriun Webcam or Camo. Just hold your phone vertically — they send the portrait feed automatically. DroidCam does NOT support portrait mode.

Can I read my micro:bit sensors?

Yes! Click "📡 Connect micro:bit" in the left sidebar. Your browser connects via Bluetooth. Accelerometer values display as an overlay on the canvas. Button A = zoom, Button B = chapter marker, tilting = laser pointer. Chrome/Edge only. NoorCast detects micro:bits advertising as either "BBC micro:bit" or "uBit".

What are the keyboard shortcuts?

R Record/Stop · L Laser · F Freeze · D Draw · S Snapshot · T Teleprompter · Z Zoom · M Marker · Q Quiz · G Grid · 1-6 Scenes · ? Random scene · Ctrl+Z Undo · Ctrl+Y Redo · Esc Stop/Exit

Full list: guide.html → Shortcuts

Are my videos private?

100%. Nothing ever leaves your computer. The recording stays in browser memory and downloads directly to your disk. No accounts, no analytics, no server. Even the fonts are self-hosted — zero network requests from JavaScript.

What output format?

WebM (VP9 + Opus) by default. MP4 (H.264 + AAC) when available (Chrome on Windows/Mac). You can choose in Settings → Recording → Output Format. Both work with VLC, YouTube, and most editors.

Is the teleprompter visible in my video?

No, never. The teleprompter is an HTML overlay shown only to you. It is NOT drawn on the canvas, so it does not appear in the recording. Same for sticky notes, the grid overlay, FPS counter, and source labels.

Which browser should I use?

Chrome or Edge (desktop, version 100+). Firefox works for basic recording but doesn't support Web Bluetooth (micro:bit). Safari is not supported. Mobile browsers are not supported (no screen capture, no multi-camera, no WebM).

What are the badges?

Local achievement trophies that unlock as you use NoorCast:

🎬 First tutorial · ⏱ 5+ minute take · 🎥 Multi-camera · 🎭 All scenes used · 🏷 5+ markers · 🤖 micro:bit connected

Stored locally in your browser. Nothing is sent anywhere.

How do I reset everything?

Settings → General → "💥 Clear all data" erases everything: badges, logo, preferences, scenes, ticker, onboarding. The app restarts fresh after a page reload. "🗑 Reset badges" only clears trophies.

Can I use NoorCast on a tablet or phone?

No. NoorCast requires desktop Chrome or Edge. Mobile browsers block screen capture, multi-camera access, and WebM recording. Use your phone as a camera source instead (via DroidCam/Iriun/Camo) connected to a desktop running NoorCast.

My recording is 0 bytes — what happened?

Check the activity log at the bottom. Try switching output format (Settings → Recording → Output Format). Make sure at least one video source is active. If using MP4, try WebM as fallback.

The video is choppy / laggy

Close other browser tabs. Reduce the canvas resolution (Settings → Recording → Stage Format). Disable overlays you're not using (clock, vignette, audio visualizer, FPS counter). Try USB instead of Wi-Fi for phone cameras.

The recorded video looks blurry / soft

Two things to check:

1. Bitrate. Settings → Enregistrement → 📶 Qualité vidéo. Default is High (8 Mbps). For motion-heavy or detailed content, bump to Ultra (12 Mbps) or Max (20 Mbps @ 60fps). Standard (4 Mbps) is OK for talking heads but not screen text.

2. Camera resolution. Cameras are now requested at 1920×1080. If your webcam can't do 1080p, the browser falls back automatically — but old USB webcams may report 1080p and deliver blurry upscaling. Check Engine.sources[i].video.videoWidth in DevTools to see what you actually got.

The download chip says "0 octets" — but VLC opens the file fine?

Chrome bug. When NoorCast streams long recordings to disk via the File System Access API, Chrome's download chip can't see the file size and shows "0 bytes" even though the file is correct.

NoorCast now shows its own toast right after the recording closes: "💾 Saved 13.4 MB to disk (ignore Chrome 0-byte chip, file is OK)". The byte count comes from our internal counter, which is accurate.

Bottom line: trust the NoorCast toast, ignore Chrome's chip.

I can't see any micro:bit when I click Connect

Most common causes (in order):

  1. Bluetooth dongle doesn't support BLE. Open chrome://bluetooth-internals → click "Start Scan". If your micro:bit doesn't appear there either, the dongle is Classic-only. Get a Bluetooth 4.0+ (BLE) USB dongle.
  2. chrome://bluetooth-internals is open. Close it — it can hold the BLE scan and prevent Web Bluetooth's pairing dialog from working.
  3. micro:bit firmware not running. The board should show a pulsing X (firmware V3.5) or a center dot (older). If it shows nothing, reflash via the firmware modal.
  4. Browser doesn't support Web Bluetooth. Chrome / Edge desktop only. Firefox and Safari don't have it.

NoorCast accepts both BBC micro:bit and uBit name prefixes — so older firmware that advertises as "uBit" still pairs.

The micro:bit shows the "tilt to fill the circle" calibration thing

That was the compass calibration prompt, fired by input.compassHeading(). NoorCast firmware V3.5 removes the compass read entirely so this prompt never fires. Reflash via Settings → micro:bit → "Flash firmware first".

The log is full of "← TP:23 ← L:0 ← S:0" spam

Old micro:bit firmware (V3.1 or earlier) sends every telemetry value every 100ms regardless of change. v0.7.212 added a JS-side log dedup that collapses identical lines into (× N), and v0.7.218 drops COMPASS: spam silently. Reload NoorCast (Ctrl+Shift+R to bypass cache).

For the actual fix: reflash to firmware V3.5 which uses change-detection — only sends a value when it crosses a threshold. Cuts BLE traffic ~10× when the board sits idle.

The COMBO xN popup is distracting in my videos

Settings → Comportement → uncheck ⚡ COMBO popup (XP bonus messages). The popup goes away immediately and never appears again until you re-enable it. Detection is also disabled so no wasted CPU per frame.

I want a different background — solid color, image, or pattern?

NoorCast supports three background layers stacked under your sources:

  1. Solid color (Settings → Apparence → 🎨 Couleur de fond) — the base layer, always visible.
  2. PNG/JPG image (Settings → Apparence → 🖼 Image de fond) — your own wallpaper, with cover/contain/stretch/tile fit and opacity slider. Max 4MB.
  3. Animated pattern (Settings → Apparence → 🖼 Background pattern) — 15 procedural patterns (Space, Matrix, Zellige, Arabesque…). Renders over the image.

All three coexist — color first, image on top, pattern over both.

How do I make the sidebar wider?

Drag the lime grip on the inner edge of either sidebar:

  • Left studio sidebar (Sources / micro:bit tabs): grip between the sidebar and the canvas. Range 180–420 px. Double-click to reset to 220px default.
  • Settings flyout sidebar: grip on its left edge. Range 240–95% viewport.

Both widths are persisted across reloads.

I can't find the global font / language flag / version date

All recently added — visible in the app:

  • UI font picker — Settings → Général → 🔤 Police de l'interface. 7 fonts (Orbitron is default).
  • Language flags — next to the language dropdown. SVG flags (FR/UK/Algerian) — work on Windows where emoji flags fail.
  • Build date — sidebar footer, next to the version badge. Hover for full timestamp.
How do I save my logo / settings to use on another computer?

Settings → Maintenance → 📦 Exporter la session. Downloads a JSON file (~50–500 KB) containing every setting, badge, brand logo (base64 inline), watermark, scene customizations, theme, etc.

On the other computer: same panel → 📥 Importer une session → pick the JSON → confirm. Page reloads with everything restored including your logo.

Recordings and snapshot files are NOT in the bundle — those are downloaded files on disk.

The Pets / Mascots dropdown — what's the difference?

Same feature, just renamed. The little animated characters (Robot, Skull, UFO, Bug, Drone…) that walk/fly across your recording were called "Pets" — now "Mascots" (🎭) since most of them aren't animals. Internally still CanvasPets.

Why is the Co-host stuck in the bottom-right corner?

Since v0.7.220 the Co-host (😊 little SVG buddy) is fully draggable and resizable. Click and drag it anywhere on the canvas. Drag a corner to resize. Position and size persist in localStorage. Bottom-right is just the default — first launch only.

I clicked a scene and nothing happened — clicked again, still nothing

Fixed in v0.7.221. The whole scene card used to be HTML5-draggable, which Chrome cancels click events on if the mouse moves even 1-2 px between mousedown and mouseup. With a jittery trackpad, kids had to click many times.

Now: the whole card registers click instantly. Drag-to-reorder is moved to the small ⋮⋮ grip in the top-left of each scene card.

Where do I find a feature by name?

Three ways to search:

  1. In-app: press Ctrl/Cmd+K (or click 🔍) for the Spotlight overlay. Searches scenes, skins, backgrounds, themes, mascots, settings, tools, sounds, badges, hotkeys. Click a result, the action runs.
  2. In the user guide: open guide.html → press Ctrl+F. The synonym cheat-sheet at the bottom of Chapter 28 lists alternative names so any search term lands on the right section.
  3. In the cheat sheet: cheatsheet.html for keyboard shortcuts.
The Settings panel doesn't close when I click outside

Fixed in v0.7.216. Bug was a CSS class-name mismatch — JS added 'show' but CSS expected 'open', so the overlay never became interactive. Reload (Ctrl+Shift+R) to pick up the fix.

Now: click the dimmed area outside the panel, OR press Esc, OR click the ✕ button — all three close the panel.

The Servo gauges or Sensor overlay show on first launch — I want a clean canvas

Fixed in v0.7.218. Both the Sensor overlay (📊) and the Servo gauge (🎛) now default to HIDDEN on first launch. Toggle on via the toolbar buttons when you actually want them.

Which firmware version is on my micro:bit?

NoorCast asks the micro:bit when it connects. Look at the small text under the Connect button in the micro:bit sidebar:

  • ✅ NoorCast V3.5 · 2026-04-17 — match · all good.
  • ⚠ NoorCast V3.1 (2026-04-12) — flash V3.5 for change-detection — outdated, reflash recommended.
  • (empty) — firmware too old to report (V3.2 or below). Reflash to V3.3+.

You can also re-query at any time from DevTools console: Sensors.sendUart('VER?').
